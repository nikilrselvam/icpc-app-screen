## ICPC Questions

#### Why ACM ICPC?
>Out of all of the CS that I have done, my biggest struggle has been with problem solving. I can understand low-level logic and high-level project structure, but I am at a loss for which algorithms to use to solve problems. Because these can be incredibly important for building efficient, robust code, I decided that I should tackle my largest weakness and develop my problem-solving skills. ICPC will give me the opportunity to see new problems and gain an intuition about how to get started and work towards devising solutions. Then, as an intern, I want to turn around and teach others what I have just learned so that they can have the same fun and excitement I do when I solve these problems.

#### What’s your favorite problem? It does not have to be an algorithm, or even CS related!
>My favorite problem is a whack-a-mole question that goes along the lines of:
>
>You are playing a whack-a-mole game with a 1D row of mole holes, which can either have a mole up in it or not. This is represented by an array of boolean values. You hit them with a hammer of width n, capable of hitting n moles in a row. Write a program that returns the maximum number of moles that you can knock with a single hit of the hammer.
>
>It's my favorite because I remember the excitement of when I was able to figure out the O(N) solution quickly, which I rarely had done before. (My answer was to make a parallel array that would track the number of moles you would hit if that index were the leftmost index of the hammer, using a linear scan through the mole array. Then, you can find the maximum value of the second array.) I love the feeling of solving a challenge, and I remembered this one vividly.
